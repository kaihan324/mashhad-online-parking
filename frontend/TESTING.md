# TESTING.md — سناریوهای تست سامانه مدیریت پارکینگ آنلاین (مشهد)

این فایل برای تست «مو به مو» قابلیت‌های پروژه (Front-End + Back-End) تهیه شده است.

## 1) پیش‌نیازها

### Back-End
```bash
cd backend
python -m venv .venv
# Windows: .venv\\Scripts\\activate
source .venv/bin/activate
pip install -r requirements.txt
python manage.py migrate
python manage.py createsuperuser
python manage.py runserver
```

### Front-End
```bash
cd frontend
npm install
npm run dev
```

> نکته: فرانت با JWT کار می‌کند. آدرس API در `frontend/src/api/client.js` تنظیم شده است (پیش‌فرض: `http://127.0.0.1:8000`).

---

## 2) نقش‌ها و دسترسی‌ها

- **user**: مشاهده پارکینگ‌ها، بررسی ظرفیت آزاد، ثبت رزرو، مشاهده/لغو رزرو، پرداخت Mock، فاکتور، اعلان‌ها، ثبت تیکت پشتیبانی، ثبت نظر.
- **parking_manager**: افزودن/ویرایش پارکینگ‌های خود، مشاهده رزروهای مرتبط با پارکینگ‌های خود، لغو رزروهای مرتبط.
- **admin**: گزارش‌ها، CRUD کاربران و نقش‌ها، CRUD پارکینگ‌ها، مشاهده همه رزروها، تایید/رد دستی رزرو، مدیریت نظرات (پاسخ ادمین)، ارسال اعلان همگانی.

## 3) تست‌های Back-End (API)

### 3.1 احراز هویت
1) ثبت‌نام
```http
POST /api/auth/register/
{ "username": "u1", "password": "12345678", "email": "u1@test.com", "phone": "09120000000" }
```
2) ورود
```http
POST /api/auth/token/
{ "username": "u1", "password": "12345678" }
```
3) پروفایل (Me)
- دریافت:
```http
GET /api/auth/me/
```
- ویرایش:
```http
PATCH /api/auth/me/
{ "phone": "09121111111" }
```

### 3.2 پارکینگ‌ها
1) لیست پارکینگ‌ها
```http
GET /api/parkings/
```
2) جستجو/فیلتر
```http
GET /api/parkings/?search=آزادی
GET /api/parkings/?min_price=10000&max_price=50000
GET /api/parkings/?min_capacity=50
GET /api/parkings/?ordering=price_per_hour
```
3) ظرفیت آزاد در بازه
```http
GET /api/parkings/<id>/availability/?start=2026-01-04T10:00:00&end=2026-01-04T12:00:00
```
4) نظرات پارکینگ
- مشاهده:
```http
GET /api/parkings/<id>/reviews/
```
- ثبت نظر:
```http
POST /api/parkings/<id>/reviews/
{ "rating": 5, "title": "عالی", "comment": "خیلی خوب بود" }
```

### 3.3 رزروها
1) ایجاد رزرو (کنترل ظرفیت و همپوشانی)
```http
POST /api/reservations/
{ "parking": 1, "car_plate": "12الف345-67", "start_time": "2026-01-04T10:00:00", "end_time": "2026-01-04T12:00:00" }
```
2) لغو رزرو
```http
POST /api/reservations/<id>/cancel/
```
3) تایید/رد دستی (ادمین/مدیر پارکینگ)
```http
POST /api/reservations/<id>/approve/
POST /api/reservations/<id>/reject/
{ "reason": "مشکل ظرفیت" }
```

### 3.4 کاربران (ادمین)
```http
GET /api/users/
POST /api/users/
PATCH /api/users/<id>/
DELETE /api/users/<id>/
```

### 3.5 اعلان‌ها
- لیست اعلان‌های کاربر:
```http
GET /api/notifications/
```
- ارسال اعلان همگانی (ادمین):
```http
POST /api/notifications/broadcast/
{ "title": "اطلاعیه", "body": "سرویس در حال بروزرسانی است" }
```

### 3.6 تیکت پشتیبانی
```http
GET /api/tickets/
POST /api/tickets/
{ "subject": "مشکل", "message": "متن مشکل" }
```

### 3.7 فاکتور
- JSON:
```http
GET /api/invoices/<reservation_id>/
```
- PDF:
```http
GET /api/invoices/<reservation_id>/pdf/
```

## 4) تست‌های Front-End (UI)

### 4.1 سناریوی کاربر (user)
1) ثبت‌نام/ورود
2) صفحه اصلی (Home): روی کارت‌های «منتخب‌های مشهد» مقدار «ظرفیت آزاد (۲ ساعت)» نمایش داده می‌شود.
3) رفتن به «پارکینگ‌ها» و باز کردن جزئیات یک پارکینگ
4) انتخاب زمان شروع + مدت و مشاهده «ظرفیت آزاد در بازه انتخابی»
5) ثبت رزرو (اگر ظرفیت آزاد > 0)
6) رفتن به «پروفایل» و مشاهده رزروها
7) انجام پرداخت (Mock) برای رزرو pending و مشاهده تغییر وضعیت
8) مشاهده «فاکتور» و دانلود PDF
9) رفتن به «اعلان‌ها» و مشاهده پیام‌ها
10) ثبت نظر برای پارکینگ از صفحه جزئیات
11) رفتن به «پشتیبانی» و ثبت یک تیکت

### 4.2 سناریوی مدیر پارکینگ (parking_manager)
> نقش کاربر را ادمین از پنل «کاربران» روی `parking_manager` تنظیم کند.

1) ورود با کاربر مدیر
2) رفتن به «پنل مدیر پارکینگ»
3) افزودن یک پارکینگ (بعد از ثبت، در لیست مدیر نمایش داده می‌شود)
4) روی همان پارکینگ «ویرایش» بزنید و ظرفیت/قیمت/آدرس را تغییر دهید
5) رزروهای مرتبط را در جدول پایین مشاهده و در صورت نیاز لغو کنید

### 4.3 سناریوی ادمین (admin)
1) ورود با ادمین
2) رفتن به «پنل ادمین»
3) تب «خلاصه»: مشاهده تعداد رزروها/کاربران/درآمد
4) تب «پارکینگ‌ها»: ایجاد/ویرایش/حذف پارکینگ
5) تب «رزروها»: تایید/رد دستی رزروهای خاص
6) تب «کاربران»: ایجاد کاربر جدید، تغییر نقش، غیرفعال کردن کاربر، حذف
7) تب «نظرات»: پاسخ ادمین به نظر کاربران
8) تب «اعلان همگانی»: ارسال یک اعلان و سپس مشاهده آن با حساب کاربر عادی
